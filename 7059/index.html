    功能介绍 通往区块链的新世界。

    **

    收录于话题

    （桌上等紫罗兰花篮，梵高）

    \

    Lisk于2016年4月6日上线交易所，其代币LSK当时价格是2.04美元，并在4天后飙升到了6.91美元，好景不长，一个月后的5月25日，价格降至0.278525美元，距离最高价格跌了24倍多。并在之后长达一年的冰冻期中，价格主要在0.10到0.25美元之间波动。但从今年4月份开始到现在涨至2017年9月25日的6.15美元左右，凸显强势回升，即使是近期整个市场低迷，也保持相对稳定高位，整体市值攀升到了数字代币市场第14位，目前（2017.9.25）估值超6.9亿美元，超过Qutum、EOS、Tenx等知名项目。

    \

    Lisk是做什么的？它有哪些特别之处？

    \

    一、简介

    **
    **

    Lisk是去中心化的区块链平台，它致力于降低区块链应用开发门槛，开发者可以用JavaScript来开发区块链应用。Lisk一些数据及资料：

    \

    主网络运行时间：2016年5月24日

    \

    区块时间：10秒

    \

    交易最终确认：6-10次

    \

    代币供应：初始1亿LSK。此外，还有打造区块的奖励，目前每个区块奖励5个LSK，之后每年减少1个LSK，直到1个LSK。目前（2017.9.25）流通的LSK超过112，713，560个。LSK可以在各种侧链应用间通用，支付各种费用，以驱动交易等。从一个账户发送LSK到另外一个账户花费10秒钟，在1-2分钟后交易不可逆。

    \

    共识算法：DPOS，允许权益人投票来解决任何共识问题。几乎所有的网络变量都可以通过这个系统来调整，包括区块大小，费用，区块间隔等。任何时候单一主体无法改变系统，需要社区达成共识才能作出改变。

    \

    项目启动获得的资金：14000个比特币（当时估值500万美元），按照今天比特币估值，团队将会有非常丰厚的资金保证在未来几年内的开发和运转费用。

    \

    二、Lisk想解决什么问题

    \

    Lisk主要解决开发者创建区块链应用的门槛问题，从目前看开发区块链应用门槛较高，而Lisk主要优势在于它的SDK基于JavaScript，它提供的框架可降低区块链应用的开发门槛。

    \

    另外，Lisk用侧链支持区块链应用，可以解决区块链臃肿问题，一般的区块链把所有的交易和应用都放在一条区块链上，易臃肿。比如Bancor、Status等ICO期间以太坊存在明显堵塞问题。与此同时，侧链与主链隔离，更具安全性。某一侧链发生问题，不会影响其他侧链或Lisk主链。

    \

    三、Lisk到底是什么

    **
    **

    1.同样是区块链应用平台，Lisk与以太坊有根本的不同

    **
    **

    大多数数字代币投资人或区块链行业的人都熟悉智能合约的概念。以太坊的理念是：智能合约将会统治世界，甚至于所有的应用会运用到某种形式的智能合约。这是以太坊系统的发展方向和基石。

    \

    而Lisk重点并不在于智能合约，当然智能合约也有可能是它未来系统的一部分，这在它的路线图中有提及。它试图通过一体化虚拟机与智能合约系统做接口，而它本身并不是智能合约系统，它是自定义的区块链系统，这是Lisk基石和理念。

    \

    它通过创建简单的用户体验和平台，便于自定义的区块链执行，可以让个人、小企业甚至大公司都可使用。这是与以太坊不同的方向。Lisk会与以太坊走完全不同的道路。

    \

    2.侧链

    **
    **

    2016年6月，The
    DAO遭受攻击，价值6000万美元的代币被盗，这件事让大家警醒，不仅仅是智能合约，包括Solidity编程语言本身也可能有问题。The
    DAO事件导致以太坊硬分叉。类似的事件，在Lisk身上不会发生。以太坊发展的是全功能和图灵完备的系统，允许开发者通过它自定义的solidity编程语言来编写智能合约。而Lisk采用的是独立的侧链技术，所有区块链应用都跑在独立的侧链上。虽然这些侧链跟主链有联系，但对主链不会产生影响。每个都是独立的区块链，有自己的共识系统、区块代表及数据库。这是以太坊和Lisk重要不同点。

    \

    Lisk上跑的每个独立应用，都有自己的侧链，自我负责。侧链上发生任何问题，应用开发者自己承担。比如侧链非法贩毒等，这些违法行为只跟侧链的应用开发者相关。以太坊等智能合约和服务都是跑在主区块链上，这极大增加了风险。类似于The
    DAO这样的事件，可能是智能合约的问题，也可能有Solidity语言自身的问题。此外，运行以太坊网络的节点在出现问题时也会有责任，例如如有人创建一个非法或恶意的智能合约，所有节点都会执行该合约。这可能产生法律责任，因为相关节点都参与了这个网络智能合约的执行。

    \

    LiskApp
    SDK是部署侧链和在上面开发区块链应用的基础框架。用JavaScript编写。开发者可以开发属于自己的区块链社交网络、即时通讯、游戏、金融应用、预测市场、交易所、存储平台、合约执行平台、分享经济、在线商店、IoT应用等，这些应用和平台都是去中心化的，无须处理复杂的共识协议或P2P网络算法等。

    \

    LiskApp
    SDK可以让开发者在Lisk上部署侧链。侧链是完全自定义的区块链，不用担心被主链或其他侧链污染。所有的规格、参数和交易类型都可以自定义。

    为了防止对区块链对垃圾攻击，须用代币做价值交换，在区块链上存储任何数据都要付出代币，也就是交易费用。在Lisk侧链，你要么用LSK代币（主链的代币），要么完全拥有自己的自定义代币。不过开发者在侧链中使用主链LSK代币时，会有限制：不同侧链间无法简单转移代币。侧链是完全独立的新的区块链。就像比特币不能发送到莱特币网络，侧链代币也不能随意转移。LSK的转账仅限于侧链在主链的账户，相同的LSK代币会复制到侧链。万一侧链发生问题，所有LSK代币仍然是安全的，能轻松恢复。当你开发一个去中介信任的应用，如开发者不想要LSK代币，也可在侧链上发行自己的独立代币。通过在区块链应用中执行小型交易系统，可实现自定义代币与LSK的兑换。

    \

    万一Lisk侧链发现了bug，出现必须硬分叉的情况，那么，只有侧链需要硬分叉，主链则保留完整，不受任何影响，也不用回滚。

    \

    现在有各种链，包括公链、私链和联盟链。Lisk上可以创建任何开发者需要的链，不管是公链、私链还是联盟链。最终来说侧链是去中心化的区块链应用数据库，可发行价值代币，是一个去中心化的网络。

    \

    3.Lisk区块链应用四个基本部分

    **
    **

    JavaScript可以让成千上万的程序员创建自定义的区块链应用，而无需学习新的语言。Lisk不是以太坊的分叉，Lisk代码是从Crypti项目发展出来的，两位创始人之前在Crypti合作已经合作了18个月。

    \

    从用户的角度，最感兴趣的是区块链应用平台。Lisk App
    SDK不仅允许侧链的部署，它也提供各种必需工具用于发展区块链应用。区块链应用不仅是在去中心化和无须中介信任的脚本，它还有如下四个基本部分：

    \

    • 共识算法（DPoS）

    • 侧链（不可逆的数据库）

    • 后端（应用逻辑）

    • 前端（应用 用户接口）

      \

    Lisk区块链应用有自己的共识算法（默认DPoS），区块链，自定义代币，后端及前端。Lisk采用了比特股的DPoS共识机制。每个LSK的持有人都可为主链代表投票，以确保网络安全运转。最多有101个激活的主链代表，他们获得最多票数，只有他们可以获得生产区块的奖励。成为打造区块代表能够获得财务上的回报。区块链应用也可利用LSK代币参与到现有的Lisk经济体系中来。Lisk
    APP SDK 让开发者不仅开发应用的后端，也可以开发前端。

    \

    4.区块链服务平台

    **
    **

    区块链服务是区块链应用的子类。它允许开发者做一些具体的单一目标的应用。比如：

    \

    • 身份服务（比如统一身份登录进入各种网站，根据需求可加密个人信息）

    • 去中心化的托管服务（在去中心化的加密云中存储图像或文件）

    • 智能合约执行服务（比如计算随机数，不被操控）

    • 预言服务（获得产品等意见或获得现实生活事件等结果）

      \

    Lisk区块链最伟大的地方在于Lisk区块链服务可以插入Lisk主链和生态系统。它们全都运行在自己的侧链上。一旦一个服务存在，其他区块链应用开发者就可利用它。比如，如果你想开发一个社交网络，并想存储用户图片，你可以插入区块链服务，或者你也可以创建桥接第三方去中心化托管服务，例如StorJ的服务。

    \

    随着时间流逝，Lisk区块链应用会变得越来越复杂，功能越来越丰富，并形成App
    生态系统。就像大脑一样，神经连接越来越多，能力越来越强。所有这些区块链服务可以融入Lisk生态中，这对提升LSK价值有利。与此同时，一旦某些区块链服务失败，只是自己失败而已，不会影响其他。

    \

    5.开源

    **
    **

    Lisk还有一个方面是开源特性。虽然有方法可以隐藏JavaScript
    代码，但大多项目并不会做出这个选择，开源才能获得信任。任何开发者把一个酷的功能整合进自己的区块链应用或服务，其他人可以拷贝、增强和整合到自己的区块链应用或服务中。

    \

    开源具有的优势：

    \

    • 一开始Lisk没有执行Solidity智能合约的字典，但把EVM整合到任何Lisk侧链是可能的。如果未来有人去做，其他人都可利用源码，要么在侧链执行智能合约，要么提供智能合约服务。开发者无需在插入以太坊网络。

      \

    • 在Lisk侧链可能需要一些时间才能有POW的共识算法，但一旦开发者开发了一个，其他能即可复用。

      \

    • 如果一个项目因为政治或财务原因失败，其他人可以复用代码并重新部署区块链，甚至可以通过编写侧链迁移把原有用户带入新的侧链。

      \

    • 去中心化应用商店

    苹果的APP store和谷歌的Play
    store都是中心化的解决方案。Lisk还有去中心化应用目录，在客户端上，每个人都可以进入。所有基于Lisk平台的区块链应用和服务必须在主链注册，通过这种方式，主链和侧链建立了连接。应用目录可以找到所有区块链应用和服务。注册时开发者可以设置唯一的区块链应用名称、描述、类别、代币、标签及下载链接。Lisk可以提供可替代的去中心化应用商店。

    \

    7.生产区块代表市场

    **
    **

    Lisk生产区块代表市场连接了区块链应用开发者和侧链区块代表。这很有必要，每个侧链都是独立的区块链，需要自己的一系列节点来确保它的安全运行。如果区块链应用开发者不使用默认的共识算法（POS、POW等），那只能自己搞定侧链矿工等，有时候可能是更好的选择。但如果区块链应用开发者使用默认的共识算法（DPoS），他将能从矿工代表市场中租用代表。这个代表租用市场只在Dpos的共识机制下有效。一些区块链应用可能会提供一些功能，这些功能在某些地区合法某些地区不合法。创业公司对此非常看重，希望保证100%的合法，这意味着侧链需要由合法管辖区的节点安全运行，这些地区区块链应用功能是合法的。节点所在地也是侧链代表市场的又一个属性。

    \

    8.Lisk不是什么

    **
    **

    以上的解释可以看出，lisk不是智能合约平台，不是在虚拟机上执行Solidity合约。Lisk区块链应用直接把他们的功能整合进他们的后端。下面是一些在Lisk区块链应用中执行具体功能的例子：

    \

    • 在区块链程序中授权

    可以依赖Lisk
    APIs。用户可以简单使用Lisk密码，可以转换成相同的区块链应用账号。

    \

    • 发送消息，写博客

    添加新的交易类型，允许添加要求范围长度的文本等。可以创建去中心化的消息服务、社交网络以及博客系统。甚至可以开发投票系统。

    \

    • 设置标志

    增加新的交易类型，设置正确或错误的状态。可以创建去中心化的IoT应用等。

    \

    • 产生随机数字

    每个在Lisk区块链应用计算中需要确定性，意味着每个节点执行区块链应用必须出来相同的结果。开发者无法使用随机的JavaScript
    功能，不得不依赖其他方法。例如计算下10个区块id的随机数字等。

    \

    四、Lisk的团队和路线图

    **
    **

    创始人Max Kordek 和Oliver
    Beddows是两位开发者，曾在Crypti合作过，两人业界名声不错，合作也紧密，区块链开发知识丰富。截止到目前为止团队所有21位成员全部公开身份，这也是一种自信表现，不像之前有些项目的成员，甚至创始人各种改名。

    \

    （Lisk团队成员合影，后排左起第三和第四位是两位创始人）

    \

    团队在资金使用上比较慎重，有打持续战的准备，预计资金至少用5年以上。Lisk有一个比较清晰的里程碑，让人印象深刻。一共五个阶段：Inception（成立）、Resilience（弹性）、Expansion（扩张）、Ascent（上升）、Eternity（永恒）。

    \

    其中inception主要是维持主链的稳定、打造矿工回报机制、启动生态基础建设、把Lisk-CII合并进Lisk
    App SDK，维持 lisk App SDK稳定。目前这一阶段基本完成。

    \

    现在进入第二阶段Resilience，这一阶段的主要是大幅提升Lisk平台，让它成为可靠的区块链应用平台，拥有优雅的核心系统、更好的Lisk
    App
    SDK，各种有用的功能，易用的用户界面，以及安全性。这个阶段在所有五个阶段中工作最为庞大，有6个里程碑，在完成这个阶段后，Lisk试图在数字资产市场独树一帜。这个阶段包括了：重新梳理品牌、守护进程、Lisk
    App SDK的模块化、基本应用功能、移动客户端及代码重构。

    \

    第三阶段是Expansion，主要增加更多应用功能、支持智能合约、支持第三方技术等。第四阶段是Ascent，一是构建完全去信任的区块链应用，任何单个实体无法关闭网络，二是随着发展，安全、优化、可扩展以及可能的共识层的变化都是可能聚焦的方面。最后是Eternity，是路线图的最后一步，最重要的是关于治理问题。比如完善去中心化的投票机制、以去中心化的方式增加建议、收集LSK持有人的意见等。到了最后的阶段，LSK将由16个知名的管理人通过多重签名账户共同管理。

    \

    从里程碑的规划看，团队是富有远见和雄心壮志，且能脚踏实地，从目前一年多的执行实施来看，基本上还是靠谱的。

    \

    五、结语

    **
    **

    目前几千万美元的估值的BTC资金可以让团队在较长时间内专注开发并逐步实现路线图。从长期投资的角度，Lisk值得考虑。不过目前它的估值稍微有点高，可以等待合适的时机进入。

    \

    如果区块链应用开发的需求越来越大，Lisk的机会越来越大。以太坊有超过200亿美元的估值，而Lisk是不是也有这样的机会？目前是6亿多美元的估值，如果一旦生态起来，达到几十亿数百亿美金的估值也并非不可能。它目前相对于以太坊的优势是所有的Lisk应用都跑在自己的侧链上，这样它有更大的机会扩展到主流的用户中。如有一天Lisk网络成为领先的区块链技术，成为创业公司创建去中心化应用的首要选择，那么，投资Lisk将会获得的非常丰厚的回报。

    \

    当然任何投资都是有风险的，任何一个小的因素都可能影响最终结局，比如说更好的竞争对手出来更好的产品，比如说区块链市场发展趋势并不需要这样的方式来解决开发者开发区块链门槛的问题，诸如此类，很多长期不可预见的因素都会产生深刻影响。

    \

    总之，Lisk是去中心化的网络，有自己的区块链，用于帮助开发者创建各种各样的自定义侧链。Lisk的价值取决于其网络生态能否成功，包括开发者、程序员及创业公司是否愿意采用。它并不是数字货币，它跟比特币不同，它并不是价值存储或支付手段。它成长的驱动力来源于其技术被大规模采用，同时在此基础上投资者的长远推动的兴趣。由于它用JavaScript作为编程语言，JavaScript是世界上最通用的编程语言之一，包括微软和谷歌都在用，这为Lisk提供了一个可能性，数量众多的程序员和创业者都可能参与到Lisk网络中来。Lisk与微软Azure达成伙伴关系，全世界的开发者可以用微软Azure进行开发、测试及部署Lisk区块链。

    \

    当然，同样也有风险。比如区块链应用本身的需求并不强烈，或者有更好的区块链平台，可以提供更好的功能。这对Lisk的价值都会有很大的影响。以太坊今天值200多个亿美元，最大原因之一是金融领域对于开发智能合约有较大需求。能够落地的需求才是支撑长远估值的核心因素。另外，在开发者没有达到一定的程度，整个Lisk生态没有其它来之前，没有更大的社区粉丝之之前，要想达到以太坊和比特币的高度还需要很长很长的路要走。（更多深度案例分析请点击下面二维码加入“蓝狐价值投资”知识星球（原小密圈））
